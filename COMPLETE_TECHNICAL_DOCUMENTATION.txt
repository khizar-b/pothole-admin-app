3. DATABASE SCHEMA
================================================================================

DATABASE: pothole_db (PostgreSQL)

TABLE: admins
├── id: Integer (Primary Key)
├── username: String(80) - UNIQUE, NOT NULL
├── password_hash: String(255) - NOT NULL (bcrypt hashed)
├── created_at: DateTime - DEFAULT utcnow()

TABLE: pothole_records
├── id: Integer (Primary Key)
├── image_name: String(255) - NOT NULL
├── timestamp: DateTime - NOT NULL (detection time)
├── bbox_x: Float - NOT NULL (bounding box X coordinate)
├── bbox_y: Float - NOT NULL (bounding box Y coordinate)
├── bbox_width: Float - NOT NULL (bounding box width in pixels)
├── bbox_height: Float - NOT NULL (bounding box height in pixels)
├── confidence_score: Float - NOT NULL (0.0-1.0 detection confidence)
├── impact_score: Float - NOT NULL (calculated 0-100 severity score)
├── bike_impact: String(20) - NOT NULL (low/medium/high for cyclists)
├── car_impact: String(20) - NOT NULL (low/medium/high for vehicles)
├── latitude: Float - NOT NULL (GPS latitude)
├── longitude: Float - NOT NULL (GPS longitude)
├── uploaded_at: DateTime - DEFAULT utcnow() (when record was uploaded)
├── uploaded_by: String(80) - NOT NULL (admin username who uploaded)

RELATIONSHIPS:
├── No foreign keys in basic schema
├── Admins can upload multiple pothole records
└── Records are timestamped for audit trail

5. IMPACT SCORE CALCULATION ALGORITHM
================================================================================

FORMULA:
impact_score = (area_normalized × 0.6) + (confidence_normalized × 0.4)

STEP-BY-STEP CALCULATION:

Step 1: Calculate Bounding Box Area
├── Formula: area = bbox_width × bbox_height
├── Unit: pixels
└── Example: 45 × 50 = 2,250 pixels²

Step 2: Normalize Area to 0-100 Scale
├── Max image area: 640 × 480 = 307,200 pixels
├── Formula: area_normalized = (area / 307,200) × 100
├── Range: 0-100
└── Example: (2,250 / 307,200) × 100 = 0.73

Step 3: Normalize Confidence Score to 0-100 Scale
├── Input range: 0.0-1.0 (from detection model)
├── Formula: confidence_normalized = confidence_score × 100
├── Range: 0-100
└── Example: 0.95 × 100 = 95

Step 4: Calculate Impact Score
├── Formula: impact_score = (area_normalized × 0.6) + (confidence_normalized × 0.4)
├── Weighting: 60% area, 40% confidence
├── Range: 0-100
└── Example: (0.73 × 0.6) + (95 × 0.4) = 0.438 + 38 = 38.438

Step 5: Determine Impact Levels
├── IF impact_score <= 30:
│   ├── bike_impact = "low"
│   └── car_impact = "low"
│
├── ELSE IF impact_score <= 60:
│   ├── bike_impact = "medium"
│   └── car_impact = "medium"
│
└── ELSE (impact_score > 60):
    ├── bike_impact = "high"
    └── car_impact = "high"

INTERPRETATION:
├── 0-30 (Low): Minor pothole, minimal safety risk
├── 31-60 (Medium): Moderate pothole, caution required
└── 61-100 (High): Severe pothole, requires urgent repair

CODE LOCATION: utils.py
├── normalize_area(bbox_width, bbox_height)
├── calculate_impact_score(area_normalized, confidence_score)
└── get_impact_level(impact_score)

7. AUTHENTICATION & SECURITY
================================================================================

PASSWORD SECURITY:
├── Hash Algorithm: bcrypt (Flask-Bcrypt)
├── Salt Rounds: 12 (configurable)
├── Hashing Workflow:
│   ├── User enters password
│   ├── Bcrypt generates salt + rounds
│   ├── Hash created: bcrypt(password + salt + rounds)
│   └── Stored in database (never store plain password)
│
├── Password Verification:
│   ├── User submits password at login
│   ├── Bcrypt compares with stored hash
│   └── Returns True/False (timing-safe comparison)
│
└── Best Practice: Minimum 6 characters enforced

JWT (JSON WEB TOKENS):
├── Library: Flask-JWT-Extended 4.6.0
├── Token Type: Bearer token
├── Payload:
│   ├── identity: username
│   ├── iat (issued at): timestamp
│   └── exp (expiration): 30 days default
│
├── Signing: HMAC-SHA256 with JWT_SECRET_KEY
├── Transmission: Authorization header
│   └── Format: Authorization: Bearer {token}
│
├── Token Validation:
│   ├── Signature verification
│   ├── Expiration check
│   └── @jwt_required() decorator on protected routes
│
└── Storage (Frontend): localStorage (client-side)

CORS (CROSS-ORIGIN RESOURCE SHARING):
├── Library: Flask-CORS 4.0.0
├── Enabled for all routes
├── Allows requests from different origins
└── Protects against CSRF attacks

AUTHORIZATION:
├── All data endpoints require JWT token
├── JWT extracted from Authorization header
├── Token decoded and user identity verified
├── Operations linked to authenticated user
└── Admin can only see/modify own uploads

API SECURITY HEADERS:
├── No password/token logging
├── HTTPS in production (recommended)
├── CORS headers configured
└── Content-Type validation on requests

9. BACKEND ARCHITECTURE
================================================================================

REQUEST FLOW:
├── 1. Request arrives at Flask app
├── 2. Routed to appropriate blueprint (auth/upload/pages)
├── 3. For API routes:
│   ├── @jwt_required() checks token (if needed)
│   ├── Token decoded, identity extracted
│   └── Request forwarded to handler function
├── 4. Handler processes request:
│   ├── Validates input
│   ├── Queries/modifies database
│   ├── Performs calculations (if needed)
│   └── Returns JSON response or redirect
└── 5. Response sent to client

DATABASE INTERACTION FLOW:
├── SQLAlchemy ORM abstracts SQL
├── Query examples:
│   ├── Admin.query.filter_by(username='admin').first()
│   ├── PotholeRecord.query.order_by(PotholeRecord.uploaded_at.desc()).paginate()
│   ├── db.session.query(PotholeRecord.car_impact, db.func.count()).group_by()
│   └── db.session.commit() - persist changes
└── Transaction handling:
    └── Rollback on error: db.session.rollback()

CSV PROCESSING PIPELINE:
├── 1. File Received
│   ├── Check file exists
│   └── Validate extension (.csv)
│
├── 2. Parse CSV
│   ├── Use pandas.read_csv()
│   ├── Load into DataFrame
│   └── Handle encoding/parsing errors
│
├── 3. Validate Schema
│   ├── Check required columns present
│   ├── List missing columns if any
│   └── Return error if invalid
│
├── 4. Process Each Row
│   ├── For each row in DataFrame:
│   │   ├── Convert data types (float, datetime)
│   │   ├── Calculate impact score:
│   │   │   ├── normalize_area()
│   │   │   ├── calculate_impact_score()
│   │   │   └── get_impact_level()
│   │   └── Create PotholeRecord object
│   ├── Track successful/failed rows
│   └── Collect error messages
│
├── 5. Database Insert
│   ├── Add all records to session
│   ├── Single commit (atomic transaction)
│   └── Rollback if any error
│
└── 6. Return Response
    ├── Success count
    ├── Total rows processed
    └── Any error messages

ERROR HANDLING:
├── Try-Except blocks on all routes
├── Specific error codes:
│   ├── 400 Bad Request (validation)
│   ├── 401 Unauthorized (auth)
│   ├── 404 Not Found (missing resource)
│   ├── 409 Conflict (duplicate username)
│   └── 500 Server Error (unexpected)
├── Database rollback on errors
└── Error messages returned as JSON

DATA VALIDATION:
├── Input validation:
│   ├── CSV column presence
│   ├── Data type conversion
│   ├── Float ranges
│   └── DateTime parsing
├── Username validation:
│   ├── Minimum 3 characters
│   └── Uniqueness check
├── Password validation:

10. AUTHENTICATION FLOW DETAILS
================================================================================

LOGIN FLOW:
├── 1. User opens /
├── 2. login.html loaded
├── 3. User enters username/password
├── 4. JavaScript captures form submission
├── 5. POST /api/auth/login with credentials
├── 6. Backend:
│   ├── Find admin by username
│   ├── Compare password hash
│   ├── If valid, generate JWT token
│   └── Return access_token
├── 7. Frontend stores token: localStorage.setItem('token', token)
├── 8. Redirect to /dashboard
└── 9. Dashboard page loads, token automatically added to API requests

PROTECTED REQUEST FLOW:
├── 1. Frontend makes API request
├── 2. Add Authorization header:
│   └── Authorization: Bearer {token}
├── 3. Request reaches backend
├── 4. @jwt_required() decorator intercepts
├── 5. Decorator:
│   ├── Extract token from header
│   ├── Decode token (verify signature)
│   ├── Check expiration
│   └── Extract identity (username)
├── 6. If valid:
│   ├── Pass identity to route handler
│   └── Handler processes request
├── 7. If invalid:
│   ├── Return 401 Unauthorized
│   └── Frontend clears token, redirects to login
└── 8. Response returned with data

REGISTRATION FLOW:
├── 1. User opens /register
├── 2. register.html loaded
├── 3. User fills form:
│   ├── Username (min 3 chars)
│   ├── Email (optional)
│   ├── Password (min 6 chars)
│   └── Confirm Password
├── 4. JavaScript validates:
│   ├── Passwords match
│   ├── Field lengths
│   └── Format validation
├── 5. POST /api/auth/register with data
├── 6. Backend:
│   ├── Validate input
│   ├── Check username uniqueness
│   ├── Hash password with bcrypt
│   ├── Create Admin record
│   ├── Commit to database
│   └── Return success message
├── 7. Frontend shows success message
├── 8. Redirect to login: /
└── 9. User can now login with new credentials

LOGOUT FLOW:
├── 1. User clicks Logout button
├── 2. JavaScript clears localStorage:
│   ├── localStorage.removeItem('token')
│   └── localStorage.removeItem('username')
├── 3. Redirect to / (login page)
├── 4. Token is gone, can't access protected routes
└── 5. User must login again

PASSWORD CHANGE FLOW:
├── 1. User on dashboard (authenticated)
├── 2. User accesses password change feature
├── 3. Enter old password + new password
├── 4. POST /api/auth/change-password
├── 5. Backend:
│   ├── Find admin by username (from token)
│   ├── Verify old password
│   ├── Hash new password
│   ├── Update record
│   ├── Commit to database
│   └── Return success
├── 6. Frontend shows confirmation
└── 7. Session continues (no re-login needed)

15. KEY FORMULAS & CALCULATIONS
================================================================================

IMPACT SCORE FORMULA:
impact_score = (area_normalized × 0.6) + (confidence_normalized × 0.4)

AREA NORMALIZATION:
area_normalized = (bbox_width × bbox_height) / 307200 × 100

CONFIDENCE NORMALIZATION:
confidence_normalized = confidence_score × 100

IMPACT LEVEL THRESHOLDS:
├── 0-30: low
├── 31-60: medium
└── 61-100: high

PAGINATION:
├── total_pages = ceil(total_records / per_page)
├── offset = (page - 1) × per_page
└── Results: [offset:offset+per_page]

├── Cause: utils.py not in project root
├── Solution: Create utils.py with impact functions
└── See: Project file structure section

Issue: CSV upload shows "UndefinedColumn" error
├── Cause: Database hasn't been migrated
├── Solution: Run migration script
└── Commands: python migrate_db.py

Issue: Login fails, token not stored
├── Cause: localStorage disabled in browser
├── Solution: Enable localStorage in browser settings
└── Check: Browser console for errors

Issue: "Unauthorized" on dashboard
├── Cause: Token expired or invalid
├── Solution: Logout and login again
└── Note: Tokens expire after 30 days

Issue: CSV file won't upload
├── Cause 1: File format not CSV
├── Solution 1: Ensure file is .csv
├── Cause 2: Required columns missing
├── Solution 2: Check sample_data_updated.csv for format

Issue: Impact scores all showing 0
├── Cause: Calculation function not called
├── Solution: Check upload.py imports utils
└── Verify: normalize_area() and calculate_impact_score() defined

Issue: Export CSV is empty
├── Cause: No records in database
├── Solution: Upload CSV first
└── Check: View Records section to verify data

Issue: Statistics not loading
├── Cause: API error or token invalid
├── Solution: Refresh page, check browser console
└── Log: Check Flask server terminal for errors

Issue: ngrok URL not accessible
├── Cause 1: ngrok not running
├── Solution 1: Start ngrok: ngrok http 5000
├── Cause 2: Flask server not running
└── Solution 2: Start Flask: python app.py
